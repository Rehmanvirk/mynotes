OOP Concepts

Classes & Objects

Classes are blueprints for objects, which encapsulate data and behavior.

A class defines properties and methods, while objects are instances created from the class. JavaScript classes (ES6) provide a structured way to model real-world entities, using constructors for initialization.

Key Points:

  • Classes use the class keyword.
  • Constructors initialize object properties with new.
  • Objects represent specific instances of a class.
  • Supports modularity and reusability.

class Person {
    constructor(name, age) {
        this.name = name;
        this.age = age;
    }
    introduce() {
        return `Hi, I'm ${this.name}, ${this.age} years old.`;
    }
}
const person = new Person("Alice", 25);
console.log(person.introduce()); // Hi, I'm Alice, 25 years old.

Encapsulation

Encapsulation bundles data and methods, restricting direct access to some components.

Encapsulation protects an object's internal state using private fields (#) and methods, exposing only necessary interfaces via getters/setters. It ensures data integrity and modularity.

Key Points:

  • Private fields prevent unauthorized access.
  • Getters/setters control data manipulation.
  • Reduces complexity and enhances maintainability.
  • JavaScript uses # for private fields (ES2020).

class BankAccount {
    #balance = 0;
    constructor(owner) {
        this.owner = owner;
    }
    deposit(amount) {
        if (amount > 0) this.#balance += amount;
        return this.#balance;
    }
    getBalance() {
        return this.#balance;
    }
}
const account = new BankAccount("Bob");
console.log(account.deposit(100)); // 100
console.log(account.getBalance()); // 100

Inheritance

Inheritance allows a class to inherit properties and methods from another class.

Inheritance creates a hierarchy where a subclass inherits from a superclass, promoting code reuse. JavaScript uses extends and super to implement inheritance.

Key Points:

  • Subclasses extend superclasses with extends.
  • super() calls the parent constructor.
  • Supports hierarchical organization.
  • Reduces redundant code.

class Animal {
    constructor(name) {
        this.name = name;
    }
    speak() {
        return `${this.name} makes a sound.`;
    }
}
class Dog extends Animal {
    constructor(name, breed) {
        super(name);
        this.breed = breed;
    }
    speak() {
        return `${this.name} barks!`;
    }
}
const dog = new Dog("Rex", "Golden Retriever");
console.log(dog.speak()); // Rex barks!

Polymorphism

Polymorphism allows objects of different classes to be treated as instances of a common superclass.

Polymorphism enables flexibility by allowing subclasses to override superclass methods, providing specific implementations while sharing a common interface.

Key Points:

  • Method overriding customizes behavior.
  • Supports dynamic method resolution.
  • Enhances code scalability.
  • Common in inheritance hierarchies.

class Shape {
    area() {
        return 0;
    }
}
class Circle extends Shape {
    constructor(radius) {
        super();
        this.radius = radius;
    }
    area() {
        return Math.PI * this.radius ** 2;
    }
}
class Rectangle extends Shape {
    constructor(width, height) {
        super();
        this.width = width;
        this.height = height;
    }
    area() {
        return this.width * this.height;
    }
}
const shapes = [new Circle(5), new Rectangle(4, 6)];
shapes.forEach(shape => console.log(shape.area())); // ~78.54, 24

Abstraction

Abstraction hides complex implementation details, exposing only necessary parts.

Abstraction simplifies interaction by hiding internal mechanics, using private fields/methods or abstract classes (simulated in JavaScript). It focuses on what an object does, not how.

Key Points:

  • Private fields (#) hide implementation.
  • Abstract classes (emulated) define interfaces.
  • Improves usability and maintainability.
  • Reduces complexity for end users.

class CoffeeMachine {
    #boiler = false;
    makeCoffee() {
        this.#heatBoiler();
        return "Coffee is ready!";
    }
    #heatBoiler() {
        this.#boiler = true;
        console.log("Boiler heated.");
    }
}
const machine = new CoffeeMachine();
console.log(machine.makeCoffee()); // Boiler heated. Coffee is ready!

Method Overriding

Method overriding allows a subclass to redefine a superclass method.

Method overriding is key to polymorphism, allowing subclasses to customize inherited methods. The super keyword can call the superclass method if needed.

Key Points:

  • Subclasses override methods for specific behavior.
  • super accesses parent methods.
  • Enables polymorphic behavior.
  • Common in inheritance chains.

class Vehicle {
    move() {
        return "The vehicle moves.";
    }
}
class Car extends Vehicle {
    move() {
        return `${super.move()} The car drives on roads.`;
    }
}
const car = new Car();
console.log(car.move()); // The vehicle moves. The car drives on roads.

Composition

Composition combines objects to form complex behaviors, favoring "has-a" relationships over inheritance.

Composition builds objects by combining simpler ones, promoting flexibility and reuse. Unlike inheritance ("is-a"), composition models "has-a" relationships, reducing tight coupling.

Key Points:

  • Promotes loose coupling.
  • Objects contain other objects.
  • More flexible than inheritance.
  • Common in modular designs.

class Engine {
    start() {
        return "Engine started.";
    }
}
class Car {
    constructor() {
        this.engine = new Engine();
    }
    drive() {
        return `${this.engine.start()} Car is driving.`;
    }
}
const car = new Car();
console.log(car.drive()); // Engine started. Car is driving.

Association

Association defines relationships between objects, where objects interact but remain independent.

Association models loose relationships, like objects referencing each other. It includes aggregation (whole-part, e.g., a department and employees) and simple associations (e.g., a teacher and student).

Key Points:

  • Objects are independent but related.
  • Includes aggregation and composition.
  • Flexible and loosely coupled.
  • Models real-world relationships.

class Teacher {
    constructor(name) {
        this.name = name;
    }
}
class Student {
    constructor(name) {
        this.name = name;
        this.teachers = [];
    }
    assignTeacher(teacher) {
        this.teachers.push(teacher);
    }
}
const teacher = new Teacher("Dr. Smith");
const student = new Student("Alice");
student.assignTeacher(teacher);
console.log(student.teachers[0].name); // Dr. Smith

OOP Exercises

Exercise 1: Student Class

Create a Student class with private grades and methods to add grades and compute average.


class Student {
    #grades = [];
    constructor(name) {
        this.name = name;
    }
    addGrade(grade) {
        if (grade >= 0 && grade <= 100) this.#grades.push(grade);
        return this.#grades;
    }
    getAverageGrade() {
        const sum = this.#grades

.reduce((a, b) => a + b, 0);
        return this.#grades.length ? (sum / this.#grades.length).toFixed(2) : 0;
    }
}
const student = new Student("Alice");
student.addGrade(85);
student.addGrade(90);
console.log(student.getAverageGrade()); // 87.50

Exercise 2: Vehicle Hierarchy

Create a Vehicle class and a Bike class that extends it, overriding a method.


class Vehicle {
    constructor(name) {
        this.name = name;
    }
    describe() {
        return `This is a ${this.name} vehicle.`;
    }
}
class Bike extends Vehicle {
    describe() {
        return `${super.describe()} It's a two-wheeler.`;
    }
}
const bike = new Bike("Harley");
console.log(bike.describe()); // This is a Harley vehicle. It's a two-wheeler.

Exercise 3: Composition

Create a Computer class with a Processor component using composition.


class Processor {
    process() {
        return "Processing data.";
    }
}
class Computer {
    constructor() {
        this.processor = new Processor();
    }
    run() {
        return `${this.processor.process()} Computer is running.`;
    }
}
const computer = new Computer();
console.log(computer.run()); // Processing data. Computer is running.
