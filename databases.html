Database Concepts

Relational Databases

Relational Databases store data in tables with rows and columns, using SQL for querying.

Relational Databases organize data into tables with defined schemas, linked by keys (primary/foreign). They ensure data integrity and support complex queries.

Key Points:

  • Uses tables with rows (records) and columns (attributes).
  • Primary keys uniquely identify records.
  • Foreign keys establish relationships.
  • Examples: MySQL, PostgreSQL, Oracle.

-- Create a table
CREATE TABLE users (
    id INT PRIMARY KEY,
    name VARCHAR(50),
    email VARCHAR(100)
);
-- Insert data
INSERT INTO users (id, name, email) VALUES (1, 'Alice', 'alice@example.com');

SQL

SQL (Structured Query Language) is used to manage and query relational databases.

SQL provides commands for creating, querying, updating, and deleting data. It includes DDL (Data Definition Language), DML (Data Manipulation Language), and DQL (Data Query Language).

Key Points:

  • DDL: CREATE, ALTER, DROP.
  • DML: INSERT, UPDATE, DELETE.
  • DQL: SELECT for querying.
  • Supports joins, aggregates, and subqueries.

-- Select users with age > 25
SELECT name, age FROM users WHERE age > 25;
-- Join tables
SELECT u.name, o.order_id
FROM users u
JOIN orders o ON u.id = o.user_id;

NoSQL

NoSQL databases handle unstructured or semi-structured data, offering flexibility.

NoSQL databases (e.g., MongoDB, Cassandra) support various data models (document, key-value, column, graph). They scale horizontally and handle large, unstructured data.

Key Points:

  • Flexible schema for dynamic data.
  • Types: Document, Key-Value, Column, Graph.
  • Scales horizontally across servers.
  • Examples: MongoDB, Redis, DynamoDB.

// MongoDB example
db.users.insertOne({
    name: "Alice",
    email: "alice@example.com"
});
// Find user
db.users.find({ name: "Alice" });

Normalization

Normalization organizes data to eliminate redundancy and ensure integrity.

Normalization divides data into tables to reduce duplication and maintain consistency, following forms (1NF, 2NF, 3NF, etc.). It optimizes storage but may increase query complexity.

Key Points:

  • 1NF: Atomic values, no repeating groups.
  • 2NF: No partial dependency.
  • 3NF: No transitive dependency.
  • Balances storage and performance.

-- Unnormalized table
CREATE TABLE orders (
    order_id INT,
    customer_name VARCHAR(50),
    customer_email VARCHAR(100),
    product VARCHAR(50)
);
-- Normalized to 1NF
CREATE TABLE customers (
    customer_id INT PRIMARY KEY,
    name VARCHAR(50),
    email VARCHAR(100)
);
CREATE TABLE orders (
    order_id INT PRIMARY KEY,
    customer_id INT,
    product VARCHAR(50),
    FOREIGN KEY (customer_id) REFERENCES customers(customer_id)
);

Transactions

Transactions ensure data integrity by grouping operations as a single unit.

Transactions follow ACID properties (Atomicity, Consistency, Isolation, Durability) to ensure reliable operations, such as bank transfers or order processing.

Key Points:

  • Atomicity: All or nothing.
  • Consistency: Maintains database rules.
  • Isolation: Transactions run independently.
  • Durability: Changes are saved.

-- Bank transfer transaction
BEGIN TRANSACTION;
UPDATE accounts SET balance = balance - 100 WHERE account_id = 1;
UPDATE accounts SET balance = balance + 100 WHERE account_id = 2;
COMMIT;

Indexing

Indexes improve query performance by enabling faster data retrieval.

Indexes create data structures (e.g., B-trees) to speed up searches on columns. They trade off storage and write speed for faster reads.

Key Points:

  • Improves SELECT query performance.
  • Slows INSERT/UPDATE due to index updates.
  • Types: B-tree, Hash, Bitmap.
  • Used on frequently queried columns.

-- Create index
CREATE INDEX idx_email ON users(email);
-- Query using index
SELECT * FROM users WHERE email = 'alice@example.com';

Joins

Joins combine data from multiple tables based on related columns.

Joins merge tables using keys (e.g., foreign keys). Types include INNER JOIN, LEFT JOIN, RIGHT JOIN, and FULL JOIN, each handling data inclusion differently.

Key Points:

  • INNER JOIN: Matching records only.
  • LEFT JOIN: All from left table, matched from right.
  • RIGHT JOIN: All from right table, matched from left.
  • FULL JOIN: All records from both tables.

-- Inner join example
SELECT u.name, o.order_id
FROM users u
INNER JOIN orders o ON u.id = o.user_id;

Database Design

Database Design involves structuring data for efficiency and scalability.

Database Design creates schemas to optimize storage, retrieval, and maintenance. It includes ER modeling, normalization, and choosing appropriate data types.

Key Points:

  • ER Diagrams model entities and relationships.
  • Normalization reduces redundancy.
  • Data types impact performance and storage.
  • Scalability and maintainability are key.

-- Example schema
CREATE TABLE employees (
    emp_id INT PRIMARY KEY,
    name VARCHAR(50),
    dept_id INT,
    FOREIGN KEY (dept_id) REFERENCES departments(dept_id)
);
CREATE TABLE departments (
    dept_id INT PRIMARY KEY,
    dept_name VARCHAR(50)
);

Database Exercises

Exercise 1: Create a Normalized Schema

Design a normalized schema for a library with books and authors.


CREATE TABLE authors (
    author_id INT PRIMARY KEY,
    name VARCHAR(50)
);
CREATE TABLE books (
    book_id INT PRIMARY KEY,
    title VARCHAR(100),
    author_id INT,
    FOREIGN KEY (author_id) REFERENCES authors(author_id)

Exercise 2: Write a SQL Query with Join

Write a query to list all employees and their department names.


SELECT e.name, d.dept_name
FROM employees e
LEFT JOIN departments d ON e.dept_id = d.dept_id;

Exercise 3: MongoDB Insert and Query

Insert a document into a MongoDB collection and query it.


// Insert document
db.products.insertOne({
    name: "Laptop",
    price: 999.99
});
// Query document
db.products.find({ price: { $gt: 500 } });
